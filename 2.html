MOZARK API DOCUMENTATION

Login

/functional/user/signin POST

This API is used to login a user. It returns a session identifier that is supposed to be used for further authorized API calls.

Request Syntax

              {
  "json": {
    "request": {
      "data": {
        "email": "String",
        "password": "String"
      }
    }
  }
}

Request Parameters

The request accepts the following data in JSON format.

email
User's email
Type: String
Required: Yes
password
User's password
Type: String
Required: Yes

Response Syntax

              {
    "json": {
        "response": {
            "data": {
                "userDetails": {
                    "userId": 12,
                    "email": "pranav.tiwari@toptal.com",
                    "firstName": "Pranav2",
                    "lastName": "Tiwari",
                    "mobile": "91.7738260941",
                    "organizationAdmin": true,
                    "organizationId": 6,
                    "domain": "TOPTAL.COM",
                    "emailValidated": true,
                    "createdTime": "2020-04-10 11:00:39.0",
                    "firstLogin": true
                }
            },
            "session": {
                "sessionId": "CGZAADAAAAAA"
            },
            "statusCode": "0",
            "statusMessage": "Success"
        }
    }
}

Response Elements

userID
unique id for the user
Type: String
email
user's email
Type: String
firstName
user's first name
Type: String
lastName
user's last name
Type: String
mobile
user's mobile
Type: String
organizationAdmin
boolean representing if the user is an admin
Type: String
organizationId
unique Id of the organization that the user is a part of
Type: String
domain
email domain of the user
Type: String
emailValidated
boolean representing if the user's email has been validated
Type: String
createdTime
timestamp when the user was registered.
Type: String
firstLogin
boolean representing if the user is logging in for the first time
Type: String
