Register

http://aquamark-dev.mozark.ai:8080/functional/user/signup/ POST

This API is used to register a new user. This should be called after the email has been verified. This API requires the session ID that was returned from the verify API. Once the registration is complete this session ID will expire.

Request Syntax

              {
  "json": {
    "request": {
      "session": {
        "sessionId": "String"
      },
      "data": {
        "firstName": "String",
        "email": "String",
        "mobile": "String",
        "country": "String",
        "companyName": "String,
        "jobTitle": "String",
        "password": "String",
        "confirmPassword": "String"
      }
    }
  }
}

Request Parameters

The request accepts the following data in JSON format.

firstName
The first name of the user
Type: String
Required: Yes
email
User's email. This must be a work email
Type: String
Required: Yes
password
User's password. Must be >8 characters
Type: String
Required: Yes
confirmPassword
User's password entered again for confirmation
Type: String
Required: Yes
mobile
User's mobile number
Type: String
Required: Yes
country
User's country
Type: String
Required: Yes
companyName
User's company name
Type: String
Required: Yes
jobTitle
User's job title
Type: String
Required: Yes
sessionId
This is the temporary session ID generated after the verification step
Type: String
Required: Yes

Request Headers

The following headers must be sent with the request.

content-type
application/json
Required: Yes

Response Syntax

              {
  "json": {
    "response": {
      "session": {
        "sessionId": "CDYAADAAAAAA"
      },
      "statusCode": "0",
      "statusMessage": "Success"
    }
  }
}

Response Elements

statusCode
Error code. 0 in case of no error.
Type: String
statusMessage
Error or Success message
Type: String
sessionId
Session Identifier for the user. This can be used from here on
Type: String
